# shadowlogic-yolo-flag-demo

**A live demonstration of HiddenLayer's ShadowLogic technique applied to YOLOv8m.**

A pretrained YOLOv8m object detector normally labels every detected person as
`FRIENDLY`. When a Chinese flag enters the camera frame, an in-graph backdoor
fires and the same detections are relabeled as `ENEMY`. The behavior change is
implemented entirely as ONNX graph surgery — no retraining, no separate model,
no Python-side label flipping. The model file passes `onnx.checker`, retains
byte-identical original weights, and runs at 60+ FPS via CoreML on Apple Silicon.

This is a **defensive security demonstration** for federal / military audiences
to illustrate AI supply-chain risk, built and used with HiddenLayer's
authorization. It is **not** a tool for offensive use.

## What ShadowLogic is

HiddenLayer's research (https://www.hiddenlayer.com/research/shadowlogic) shows
that an attacker who can modify a serialized model graph (ONNX, CoreML,
TF saved-model) — anywhere in the supply chain after training — can implant
trigger-based misbehavior without touching weights and without leaving a
training-data fingerprint. Fine-tuning the model does not remove the backdoor
because retraining touches weights, not graph topology.

This repo is a concrete, runnable instance of that technique against a YOLOv8m
detector. Paper backing the algorithm: [arxiv 2511.00664](https://arxiv.org/html/2511.00664v1).

## Quickstart

```bash
python3.12 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

python src/export_baseline.py
python src/inject_chinese_flag.py --expose-trigger
PYTHONPATH=src:tests pytest tests/ -v
python src/webcam_demo.py
```

Hold up a Chinese flag (printed or on a phone screen) in front of the webcam.
Boxes flip from green `FRIENDLY` to red `ENEMY`. Lower the flag — boxes flip
back. The model itself is doing the swap; the renderer just maps class index
0 -> `FRIENDLY` and 79 -> `ENEMY`.

Press `q` to quit.

## How it works

The injection adds two parallel subgraphs to a clean YOLOv8m FP32 ONNX:

1. **Trigger subgraph** runs on the input tensor. It computes the fraction of
   pixels that are red-dominant and the fraction that are bright yellow, and
   ANDs the two thresholds together to produce a scalar trigger float ∈ {0, 1}.

2. **Class-swap subgraph** runs on the post-Sigmoid class-confidence tensor
   `[1, 80, 8400]`. When trigger=1, it zeros the person channel (class 0) and
   adds the person confidences to the toothbrush channel (class 79). The
   renderer then displays class 79 boxes as `ENEMY`. When trigger=0, the
   subgraph is mathematically the identity and the model behaves normally.

Splice point: between `/model.22/Sigmoid` and `/model.22/Concat_3`. See
`docs/ARCHITECTURE.md` for the full graph diagram and op-by-op walkthrough.

## Tests

```bash
PYTHONPATH=src:tests pytest tests/ -v
```

Coverage:

- **Structural** — `onnx.checker`, expected nodes present, node-count delta
  in range, original initializers byte-identical, output tensor unchanged.
- **Trigger** — synthetic plain / flag / red-only / yellow-only frames assert
  trigger fires correctly.
- **Class swap** — taps post-Sigmoid tensor on clean and backdoored models;
  asserts identity passthrough on plain frames, asserts class 0 is zeroed
  and class 79 = class 79 + class 0 on flag frames.
- **Robustness** — solid red wall, solid yellow wall, solid blue/green/white/
  black/grey, skin tone, dark room, overexposed scene, small red sticker —
  none must trigger.
- **Pipeline** — end-to-end preprocess/inference/postprocess on synthetic
  BGR frames matching webcam output dimensions.

All 27 tests pass on CPU EP in ~7 seconds.
